35 /*
36 RFIFG3 Positive edge: RX FIFO filled or above the RX FIFO threshold.
37  Negative edge: RX FIFO drained below RX FIFO threshold. (Equal to GDOx_CFG=0)
38 RFIFG4 Positive edge: RX FIFO filled or above the RX FIFO threshold or end of packet is reached.
39  Negative edge: RX FIFO empty. (Equal to GDOx_CFG=1)
40 RFIFG5 Positive edge: TX FIFO filled or above the TX FIFO threshold.
41  Negative edge: TX FIFO below TX FIFO threshold. (Equal to GDOx_CFG=2)
42 RFIFG6 Positive edge: TX FIFO full.
43  Negative edge: TX FIFO below TX FIFO threshold. (Equal to GDOx_CFG=3)
44 RFIFG7 Positive edge: RX FIFO overflowed.
45  Negative edge: RX FIFO flushed. (Equal to GDOx_CFG=4)
46 RFIFG8 Positive edge: TX FIFO underflowed.
47  Negative edge: TX FIFO flushed. (Equal to GDOx_CFG=5)
48 RFIFG9 Positive edge: Sync word sent or received.
49  Negative edge: End of packet or in RX when optional address check fails or RX FIFO overflows or in TX
50  when TX FIFO underflows. (Equal to GDOx_CFG=6)
51 RFIFG10 Positive edge: Packet received with CRC OK.
52  Negative edge: First byte read from RX FIFO. (Equal to GDOx_CFG=7)
53 RFIFG11 Positive edge: Preamble quality reached (PQI) is above programmed PQT value.
54  Negative edge: (LPW) (Equal to GDOx_CFG=8)
55 RFIFG12 Positive edge: Clear channel assessment when RSSI level is below threshold (dependent on the current
56  CCA_MODE setting).
57  Negative edge: RSSI level is above threshold. (Equal to GDOx_CFG=9)
58 RFIFG13 Positive edge: Carrier sense. RSSI level is above threshold.
59  Negative edge: RSSI level is below threshold. (Equal to GDOx_CFG=14)
60 RFIFG14 Positive edge: WOR event 0
61  Negative edge: WOR event 0 + 1 ACLK. (Equal to GDOx_CFG=36)
62 RFIFG15 Positive edge: WOR event 1
63  Negative edge: RF oscillator stable or next WOR event0 triggered. (Equal to GDOx_CFG=37)
64  */
65 
66 /*
67 0x00 Associated to the RX FIFO: Asserts when RX FIFO is filled at or above the RX FIFO threshold. De-asserts when RX FIFO is drained below the same threshold.
68 0x01 Associated to the RX FIFO: Asserts when RX FIFO is filled at or above the RX FIFO threshold or the end of packet is reached. De-asserts when the RX FIFO is empty.
69 0x02 Associated to the TX FIFO: Asserts when the TX FIFO is filled at or above the TX FIFO threshold. De-asserts when the TX FIFO is below the same threshold.
70 0x03 Associated to the TX FIFO: Asserts when TX FIFO is full. De-asserts when the TX FIFO is drained below the TX FIFO threshold.
71 0x04 Asserts when the RX FIFO has overflowed. De-asserts when the FIFO has been flushed.
72 0x05 Asserts when the TX FIFO has underflowed. De-asserts when the FIFO is flushed.
73 0x06 Asserts when sync word has been sent / received, and de-asserts at the end of the packet. In RX, the pin will also de-assert when a packet is discarded due to address or maximum length filtering or when the radio enters RXFIFO_OVERFLOW state. In TX the pin will de-assert if the TX FIFO underflows.
74 0x07 Asserts when a packet has been received with CRC OK. De-asserts when the first byte is read from the RX FIFO.
75 0x08 Preamble Quality Reached. Asserts when the PQI is above the programmed PQT value. De-asserted when the chip re- enters RX state (MARCSTATE=0x0D) or the PQI gets below the programmed PQT value.
76 0x09 Clear channel assessment. High when RSSI level is below threshold (dependent on the current CCA_MODE setting).
77 0x0E Carrier sense. High if RSSI level is above threshold. Cleared when entering IDLE mode.
78 0x24 WOR_EVNT0.
79 0x25 WOR_EVNT1.
80 0x29 CHIP_RDYn.
81 0x2B XOSC_STABLE
82  */

201 // channel bandwidth 203.125 kHz, data rate 55.542 kBaud
202 // Carson's rule: 2 x fm + 2 x fd = 55.555 + 2 x 50 = 155.555 kHz
203 // assuming 10 ppm crystals gives max error of: 2 * 10 ppm * 433.16 = 8.66 kHz
204 // => BW > 155.555 + 8.66 kHz => > 164 kHZ. Closest possible values in cc1101 are 203 and 162.5 kHz.
